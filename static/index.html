SYSTEM

Built like software infrastructure, not a novelty chatbot.

Trust comes from visible boundaries: what gets routed, what gets executed, what gets remembered, and what still needs a person.

Good automation feels calm because the structure is explicit. That clarity is what makes the product feel premium. Not effects. Not slogans.
operating_model
  1. Classify Separate finance, scheduling, reminders, and delivery before execution starts.
  2. Execute Run only the channels and tools needed for the task in front of the user.
  3. Deliver Return a brief, a review flag, or a resolved state instead of system noise.
routing

Intent is routed before tools run.

Expense logging, booking, reminders, reporting, and inbox work do not compete in a single fuzzy loop.

memory

Preferences stay useful.

Recurring habits and preferences are retained so the product feels personal without becoming messy.

review

Risky steps wait for confirmation.

That boundary makes the experience more trustworthy than pretending every action should run unattended.
